# Краткие заметки для отчёта

## Задача 1
Использованы: `requests`, `BeautifulSoup`.

Сложности:
- на странице SQLite CVE опубликованы на одной общей странице, а не на отдельных карточках;
- часть записей может быть отмечена как duplicate или not a bug in SQLite;
- у некоторых записей дата может отсутствовать прямо в строке.

Решение:
- все CVE-ID ищутся по шаблону `CVE-YYYY-NNNN...`;
- дата извлекается из текста строки таблицы;
- ссылка `vendor_release_url` для SQLite одинакова для всех записей и указывает на официальную страницу `https://www.sqlite.org/cves.html`.

## Задача 2
Использованы: `requests`, официальный CVE источник (`cveawg.mitre.org` / fallback `cvelistV5`), официальный CWE REST API.

Сложности:
- структура CVE JSON может отличаться у разных записей;
- не у всех CVE присутствуют CVSS, CPE или CWE;
- данные по CWE в самом CVE record могут быть неполными.

Решение:
- сделан универсальный разбор `containers.cna` и `containers.adp`;
- для CVSS собираются все доступные версии;
- CPE собираются рекурсивным поиском по JSON;
- CWE-ID берутся из `problemTypes`, а имя и описание подтягиваются из CWE REST API.

## Задача 3
Использованы: `xml.etree.ElementTree`, `xml.dom.minidom`.

Сложности:
- нужно было сохранить вложенность и соблюсти требуемый формат дочерних элементов.

Решение:
- поля 1 уровня записываются отдельными тегами;
- `cvss_list`, `cpe_list`, `cwe` преобразуются в дочерние элементы нужного формата.

## Задача 4
Использованы: `jsonschema`.

Сложности:
- по заданию нужны обязательные поля и минимум один дочерний элемент;
- в реальных CVE такие данные могут отсутствовать.

Решение:
- схема сделана строгой;
- если отдельные записи не проходят валидацию, в отчёте нужно указать, какие именно данные отсутствуют в источнике и что для прохождения валидации понадобилось бы их искусственно заполнять.

## Задача 5
Использованы: PostgreSQL, Docker, `psycopg`.

Сложности:
- данные нужно привести к 3НФ;
- одна уязвимость может иметь много CVSS, CPE и CWE.

Решение:
- выделены отдельные таблицы `vulnerabilities`, `cvss_metrics`, `cpes`, `cwes`;
- связи many-to-many реализованы через `vulnerability_cpes` и `vulnerability_cwes`.
